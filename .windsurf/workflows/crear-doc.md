---
description: Proceso de Creación de Documentación Robusta
---

1. **Fase de Descubrimiento (Deep Dive)**:
   - Analizar el código fuente relacionado para entender la lógica interna.
   - Identificar a quién va dirigido (Junior, Senior, Stakeholder).
   - Determinar el cuadrante Diátaxis adecuado.

2. **Diseño de Estructura (Blueprint)**:
   - Proponer un índice detallado (H1 a H3).
   - Explicar qué se cubrirá en cada sección para asegurar que no falten ideas clave.
   - Solicitar feedback al usuario sobre el alcance.

3. **Redacción Expandida**:
   - Escribir contenido rico en detalles pero fácil de leer.
   - Incluir ejemplos de uso real y manejo de errores.
   - Asegurar que cada párrafo aporte valor al objetivo del documento.

4. **Revisión y Pulido**:
   - Verificar enlaces, formatos Markdown y exactitud técnica.
